================================================================================
 BASE DE DONNÉES - MIGRATION REQUISE
================================================================================

IMPORTANT : Cette mise à jour nécessite l'exécution d'un script SQL.

Script à exécuter : Database\add_premier_connexion_column.sql

Modifications apportées :
  1. Ajout colonne "premier_connexion" (BOOLEAN)
     - TRUE si l'utilisateur doit changer son mot de passe
     - FALSE si déjà changé

  2. Ajout colonne "mot_de_passe_par_defaut" (VARCHAR(20))
     - Stocke le mot de passe temporaire pour référence admin
     - NULL après changement

  3. Mise à jour des utilisateurs existants
     - Marqués comme "premier_connexion = FALSE"
     - Pas de changement obligatoire pour les comptes existants

MÉTHODES D'EXÉCUTION :

A) Via MySQL Workbench (Recommandé) :
   1. Ouvrir MySQL Workbench
   2. Se connecter à votre base de données
   3. File → Open SQL Script
   4. Sélectionner : Database\add_premier_connexion_column.sql
   5. Cliquer Execute (Ctrl+Shift+Enter)

B) Via Ligne de Commande :
   mysql -u utilisateur -p nom_base < Database\add_premier_connexion_column.sql

VÉRIFICATION :
  SHOW COLUMNS FROM utilisateurs LIKE 'premier_connexion';
  SHOW COLUMNS FROM utilisateurs LIKE 'mot_de_passe_par_defaut';

 WORKFLOW UTILISATEUR
================================================================================

1. CRÉATION D'UN NOUVEL UTILISATEUR (Admin)

   Étape 1 : Admin se connecte
   Étape 2 : Administration → Gestion des Utilisateurs → Ajouter
   Étape 3 : Remplir les informations (nom, email, rôles)
   Étape 4 : Cliquer "Enregistrer"

   Résultat : Message affiche "Mot de passe par défaut : RHPlus2026!"
             Admin communique ce mot de passe à l'utilisateur

2. PREMIÈRE CONNEXION (Nouvel Utilisateur)

   Étape 1 : Utilisateur se connecte avec "RHPlus2026!"
   Étape 2 : Message : "Première connexion détectée"
   Étape 3 : Formulaire de changement s'affiche automatiquement
   Étape 4 : Utilisateur saisit son nouveau mot de passe
   Étape 5 : Validation stricte :
             - Minimum 8 caractères
             - Au moins 1 majuscule, 1 minuscule, 1 chiffre, 1 spécial
             - Différent de "RHPlus2026!"
   Étape 6 : Déconnexion automatique
   Étape 7 : Reconnexion avec le nouveau mot de passe

3. CONNEXIONS SUIVANTES

   Utilisateur se connecte normalement avec son mot de passe personnel
   Pas de formulaire de changement

 COMMANDES SQL UTILES
================================================================================

LISTER UTILISATEURS EN ATTENTE DE CHANGEMENT :
  SELECT nom_utilisateur, nom_complet, date_creation
  FROM utilisateurs
  WHERE premier_connexion = TRUE;

RÉINITIALISER UN UTILISATEUR (forcer changement) :
  UPDATE utilisateurs
  SET premier_connexion = TRUE,
      mot_de_passe_hash = SHA2('RHPlus2026!', 256),
      mot_de_passe_par_defaut = 'RHPlus2026!'
  WHERE nom_utilisateur = 'nom_user';

DÉSACTIVER LE CHANGEMENT OBLIGATOIRE :
  UPDATE utilisateurs
  SET premier_connexion = FALSE,
      mot_de_passe_par_defaut = NULL
  WHERE nom_utilisateur = 'nom_user';

VÉRIFIER L'ÉTAT D'UN UTILISATEUR :
  SELECT nom_utilisateur, premier_connexion, mot_de_passe_par_defaut, actif
  FROM utilisateurs
  WHERE nom_utilisateur = 'nom_user';

================================================================================
 DÉPANNAGE
================================================================================

PROBLÈME : Erreur SQL lors de l'exécution du script
SOLUTION : Vérifier que vous utilisez MySQL 5.7+ et que l'utilisateur a les
           droits ALTER TABLE. Consulter CORRECTION_SQL_MIGRATION.md

PROBLÈME : Formulaire de changement ne s'affiche pas
SOLUTION : Vérifier que le script SQL a été exécuté et que
           premier_connexion = TRUE dans la base

PROBLÈME : Admin RH voit encore les menus système
SOLUTION : Vérifier que l'application a été recompilée et redémarrée
           Consulter TEST_RESTRICTION_ADMIN_RH.md

PROBLÈME : Utilisateur bloqué après changement de mot de passe
SOLUTION : Vérifier que premier_connexion est passé à FALSE
           Si bloqué, exécuter : UPDATE utilisateurs SET premier_connexion = FALSE WHERE...
